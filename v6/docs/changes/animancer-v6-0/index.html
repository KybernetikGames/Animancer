Animancer v6.0

This version is currently under development and is available for testing.

Planned Release 2020-12-??

This page was last updated on 2020-11-06

Upgrade Guide

When upgrading to a newer version of Animancer, you must delete any previous version from your project first.

  • See the Animancer v5.0 Upgrade Guide first if you are upgrading from a version older than that.
  • The Animancer.FSM system now uses a new StateChange system:
    • The following types no longer require a generic argument: IState, StateBehaviour, DelegateState.
    • IState.CanEnterState and CanExitState are now properties.
    • InterruptManagement.CreatureState.CanExitState has comments explaining how to access the StateChange details (will be documented later).

Major Features

  • Added a new StateChange system to the Animancer.FSM: #73
    • StateChange and KeyChange now provide static access to the previous/next state during a change.
    • Added comments to InterruptManagement.CreatureState.CanExitState explaining the different ways of accessing the state change.
    • Removed the generic TState argument from IState since CanEnterState and CanExitState don't need to take the previous/next state as parameters anymore.
    • Removed the TState argument from StateBehaviour and DelegateState since IState no longer requires it.
    • Removed the DelegateState constructor since object initialiser syntax is clearer anyway.
    • Changed IState.CanEnterState and CanExitState to properties since they no longer require a parameter.
    • Added FSM.State as an abstract class that implements IState using virtual methods.
    • Added Game Manager example as the first in the State Machines group which demonstrates the Animancer.FSM without StateBehaviours in comparison to an enum based system.
    • Removed the InterruptManagement.Creature script and modified its CreatureState to inherit from Creatures.CreatureState now that StateBehaviour is no longer generic.
  • Added AnimancerToolsWindow which can be opened via Window/Animation/Animancer Tools and has 5 panels:
    • Move Sprite Borders: allows you to add or subtract from the borders of Sprites.
    • Rename Sprites: allows you to bulk-rename Sprites.
    • Generate Sprite Animations allows you to generate animations from Sprites based on their names. See the Sprites page for more information (all these panels will be documented in more detail).
    • Remap Sprite Animation: allows you to change which Sprites an animation uses.
    • Remap Animation Bindings: allows you to change the Hierarchy paths used by an animation. This is particularly useful for bone animations if you want to rename the bones in a model.
  • Mixer Transitions can now use either AnimationClips or any ITransition that inherits from UnityEngine.Object (including Transition Assets) for their children, meaning that you can now fully set up Nested Mixers using transitions instead of in code.
    • Renamed ManualMixerState.Transition.Clips to States and changed it from an AnimationClip[] to an Object[].
    • Added several AnimancerUtilities.TryGet... methods that take an Object and output a particular value from it as an AnimationClip or the corresponding value from an ITransition.

Minor Features

  • Added AnimancerEvent.Sequence.RemoveCallback. #77
  • Added Initialise 8 Directions function to the Thresholds menu for MixerState.Transition2D.
  • Added ITransitionDetailed.AverageAngularSpeed and AverageVelocity.

Changes

  • Renamed ControllerState.Parameter to ParameterID and cleaned up its usage:
    • It is now a readonly struct.
    • Added a constructor that takes both a name and hash (but doesn't verify them).
    • Float1ControllerState now exposes its ParameterID directly instead of having separate properties for the name and hash (same for the 2 and 3 variants).
  • Removed the Reset Transition Context Menu Function.
  • Moved AnimancerEvent.Sequence.Serializable.DummyCallback out to AnimancerEvent.Sequence.

Fixes

  • Implemented a workaround for the Serialized Array Initialization Bug for Transitions: if all the fields of a transition are at the default values of their type (i.e. Fade Duration and Speed at 0 rather than the 0.25 and 1 set by their field initialisers) then assigning its main object (i.e. the AnimationClip for a ClipState.Transition) will initialise its values properly.
  • Fixed AnimationBindings.GetBindings to allow gathering multiple times per frame and only disable it for GUI calls.
  • Fixed the TransitionPreviewWindow models list to automatically remove duplicates.
  • Fixed AnimancerLayer.Play(state, fadeDuration, fadeMode) to set the Weight = 1 if it redirects to Play(state).
  • Fixed TransitionPreviewWindow to not cause errors when it instantiates a copy of an object that has [RequireComponent] attributes on its components which create a circular reference (A requires B and B requires A). It doesn't seem to be possible to destroy such components so it now just disables all undesirable components instead of destroying them.
    • Removed AnimancerEditorUtilities.TopologicalSort since it is no longer needed.
  • Fixed the keyed StateMachine to properly register the default state provided in the constructor.
