01 Simple Playing

Assets/Plugins/Animancer/Examples/01 Basics/01 Simple Playing

This example demonstrates how to use Animancer to play animations and how to register a callback to be notified when an animation finishes playing.

It consists of one very simple script called PlayAnimationOnClick which starts with an idle animation and performs an action when the user clicks the mouse, then returns to idle.

  • It references an AnimancerComponent instead of referencing the Animator like you would without Animancer. Most operations that interact with Animancer are done via that component.
  • It also references two AnimationClips which contain the actual animation data that defines how the model will move over time.
  • These references are all serialized, meaning they can be assigned using the Inspector.
  • The scene contains two copies of the same model with the same scripts attached, but different animations assigned to show how a simple script can be reused for different purposes.
  • The Simple and End Events example is an extension of this one which shows how to use Animation Events to trigger a function at a specific time during an animation to hit a ball and also how to adjust the transition back to idle to look more natural.

Tutorial

This tutorial takes you through a step by step process of implementing a PlayAnimationOnClick script and setting up a scene to see it in action. Since this is the first tutorial, it covers the basics more thoroughly than the rest. Click here to see the full script.

using Animancer;
using UnityEngine;

public sealed class PlayAnimationOnClick : MonoBehaviour
{
    [SerializeField]
    private AnimancerComponent _Animancer;

    [SerializeField]
    private AnimationClip _Idle;

    [SerializeField]
    private AnimationClip _Action;

    private void OnEnable()
    {
        _Animancer.Play(_Idle);
    }

    private void Update()
    {
        if (Input.GetMouseButtonDown(0))
        {
            var state = _Animancer.CrossFade(_Action);
            state.OnEnd = OnActionEnd;
        }
    }

    private void OnActionEnd()
    {
        _Animancer.CrossFade(_Idle);
    }
}

Start by following the Basic Scene Setup instructions to create a new scene with the character in it.

Idle Animation

Now let's get the idle animation working.

Create a new script by right clicking in the Project window and selecting Create/C# Script. Call it PlayAnimationOnClickTutorial and open it up in your IDE.

The first thing the script needs is a reference to an AnimancerComponent which we want to assign using the Inspector. This could be done with a simple public AnimancerComponent animancer, but being public would mean that other scripts can change the reference when we aren't expecting it, so it's better to use:

[SerializeField]
private AnimancerComponent _Animancer;

Since AnimancerComponent is in the Animancer namespace, we either need to put using Animancer; at the top of the script or we need to specify the full name of the type as Animancer.AnimancerComponent.

We also need a reference to the idle animation, so let's add another field:

[SerializeField]
private AnimationClip _Idle;

Then we can just tell it to play that animation on startup:

private void OnEnable()
{
    _Animancer.Play(_Idle);
}

The full script now looks like this:

using Animancer;
using UnityEngine;

public sealed class PlayAnimationOnClickTutorial : MonoBehaviour
{
    [SerializeField]
    private AnimancerComponent _Animancer;

    [SerializeField]
    private AnimationClip _Idle;
    
    private void OnEnable()
    {
        _Animancer.Play(_Idle);
    }
}

You will notice that this script doesn't currently contain either of the Start or Update methods which Unity puts in new scripts by default, so instead of needing to manually remove them all the time you might consider replacing your Default Script Template.

Let's save the script and go back to Unity to see it in action.

  1. Select your character in the Hierarchy window (the root object with the Animator component).
  2. Make sure the Apply Root Motion toggle is deselected since we don't want any movement in this example.
  3. Drag the script we just created from the Project window onto the bottom of the Inspector to add it (or you can use the Add Component button).
  4. Drag and drop the AnimancerComponent into the Animancer field in that component.
  5. Use the circular icon next to the Idle field to open the Object Picker window to find the idle animation you want to use. The original example uses Humanoid-GolfSwingReady.

Click here if you're wondering why some things look a little different from Unity's defaults in that video.

It's because of a plugin called Inspector Gadgets (also made by Kybernetik like Animancer).

It replaces the default Transform Inspector to add features like the [C][P][S] buttons on the right to Copy and Paste values between objects, and Snap them to the grid or the [=] button on the left which toggles the Scale field from a single value to the regular x/y/z vector field.

Inspector Gadgets Pro also adds [H][S][A] buttons to the object reference fields (on the right of the PlayAnimationOnClickTutorial script) while they are null to allow you to find a reference in a single click by searching the Hierarchy (parents and children of the current object), the current Scene, or all Assets in the project. So we could have just clicked the [H] to get the AnimancerController reference from the same object instead of dragging and dropping it. Note how the Idle field didn't have a [H] button because AnimationClips are assets, not scene objects.

Now if we press Play, the model will be playing the idle animation we gave it.

Don't forget to exit Play Mode once you are done looking.

Golf Club

That pose looks a bit funny without anything to hold so let's give him a golf club.

  1. Expand the character's hierarchy until you reach the Holder.R bone (an extra bone which is a child of the right hand in a good position for holding objects). Note that if you hold Alt when you expand or collapse something, it will do the same for all of that object's children as well.
  2. Go to the Assets/Plugins/Animancer/Examples/Art/Golf folder.
  3. Drag the GolfClub prefab from the Project window onto the Holder.R bone in the Hierarchy window to instantiate the prefab as a child of that bone.

Now he will be holding the golf club when we press Play.

Don't forget to exit Play Mode once you are done looking.

Swing Animation

A golf club won't do much good if he can't swing it, so let's get on that.

First we need to add another AnimationClip field to the script. Just copy the _Idle field and rename it to _Action.

Then we want to know when the player clicks the mouse, so let's make an Update method to check that:

private void Update()
{
    if (Input.GetMouseButtonDown(0))
    {
    }
}

Inside that if block we want to play the _Action animation, so we can just copy the contents of the OnEnable method and change the parameter.

The full script now looks like this:

using Animancer;
using UnityEngine;

public sealed class PlayAnimationOnClickTutorial : MonoBehaviour
{
    [SerializeField]
    private AnimancerComponent _Animancer;

    [SerializeField]
    private AnimationClip _Idle;
    
    [SerializeField]
    private AnimationClip _Action;
    
    private void OnEnable()
    {
        _Animancer.Play(_Idle);
    }

    private void Update()
    {
        if (Input.GetMouseButtonDown(0))
        {
            _Animancer.Play(_Action);
        }
    }
}

Save the script again and go back to Unity.

Before it will do anything, we need to give it the swing animation. Just select the character and assign the Action animation in the Inspector the same way we assigned the Idle. The original example uses the Humanoid-GolfSwing animation.

Now when we enter Play Mode we can click to trigger the swing animation.

Don't forget to exit Play Mode once you are done looking.

On End Callback

The character now has an obvious problem: he can swing like a champ, but then he gets delayed stagefright and freezes in place. Fortunately there's a simple cure for that if we go back to the script.

All we want to do is wait until the animation is done then return to idle so let's first create a method to be called:

private void OnActionEnd()
{
    _Animancer.Play(_Idle);
}

Then we can grab the AnimancerState returned by Play and assign that method as a callback to its OnEnd event:

var state = _Animancer.Play(_Action);
state.OnEnd = OnActionEnd;

There are a couple of different ways we could have done that which are listed on the AnimancerState.OnEnd page. The Playing Animations page also explains some alternative approaches instead of using the OnEnd callback.

If we take another look at it in Play Mode we can see the character return to idle after the swing so we can click to swing again.

Don't forget to exit Play Mode once you are done looking.

Smooth Transitions

There's still an obvious problem with it though: when the swing ends, the character snaps instantly back to idle instead of moving smoothly from where the swing ended. This is easily solved using Cross Fading. The next example Playing and Fading goes into more detail on the subject, but let's give it a quick try here.

All you need to do is change Play to CrossFade.

The final script looks like this:

using Animancer;
using UnityEngine;

public sealed class PlayAnimationOnClickTutorial : MonoBehaviour
{
    [SerializeField]
    private AnimancerComponent _Animancer;

    [SerializeField]
    private AnimationClip _Idle;

    [SerializeField]
    private AnimationClip _Action;

    private void OnEnable()
    {
        _Animancer.Play(_Idle);
    }

    private void Update()
    {
        if (Input.GetMouseButtonDown(0))
        {
            var state = _Animancer.CrossFade(_Action);
            state.OnEnd = OnActionEnd;
        }
    }

    private void OnActionEnd()
    {
        _Animancer.CrossFade(_Idle);
    }
}

And here's what it looks like in Play Mode:

By default, a CrossFade will take 0.3 seconds, but if you have Animancer Pro you could specify the duration after the animation like so _Animancer.CrossFade(_Idle, duration);. If you only have Animancer Lite you can still try that in the Unity Editor to see what it looks like, but it will always use the default in a runtime build.

That's all for this first tutorial. The next one goes into more detail about the differences between Play, CrossFade, and the ways they are commonly used.
