The Animancer Inspector

The inspector for the base AnimancerController component only has a single field: a reference to the Animator component you want it to control. Once that reference is assigned, it will automatically collapse the inspector of the Animator and show its relevant fields beneath the reference.

What it looks like
The collapsed Animator component. You can freely expand it again if you want to see it.
On top of simply saving a bit of screen space, embedding the Animator's fields allows Animancer to display contextual warnings for it:
  • Having an AnimatorController assigned while using an AnimancerController component will apply the output from Animancer to the model, however Unity will still execute it's state machine in the background, which wastes processing time unless you're doing it intentionally.
  • Changing the Update Mode to or from AnimatorUpdateMode.AnimatePhysics at runtime will have no effect. See Update Modes for more details.
  • You can access this documentation via the blue [?] help icon in the top right.
The context menu of the AnimancerController component has various useful functions.
In Play Mode, the inspector shows the details of all animation layers and states.
  • Each state can be expanded to show more of its details with manual controls for debugging.
  • Each of the controls directly correlates to a property with the same name in the AnimancerState class which can be accessed by code.
Each individual state has its own context menu which displays more of its details and several functions specific to it.
