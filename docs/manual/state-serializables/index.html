State Serializables

In addition to the Play and CrossFade methods, the AnimancerComponent.Transition method can also be used to start animations. It takes an IAnimancerTransition parameter which contains the details necessary to create an AnimancerState and choose which of the Play or CrossFade methods to actually use.

That interface is implemented by a series of serializable classes corresponding to each type of AnimancerState which allow you to specify animation details in the Inspector so they can be edited as part of the scene (or prefab) rather than being hard coded. These classes are all called Serializable (or something similar) and are nested inside the type of state they create. For example, a ClipState.Serializable will create a ClipState (to play a single AnimationClip).

Unfortunately, the tool used to generate this site (Wyam 2.2.4) does not currently support nested types with identical names so their API documentation hasn't been generated properly.

Basic Serializables

Type Details
AnimancerState
.Serializable<TState>		 abstract base class for all other serializables. Only has a FadeDuration field.
ClipState
.Serializable

Inherits from AnimancerState.Serializable<ClipState> to add an AnimationClip, Speed, and optional StartTime which can set either AnimancerState.Time or AnimancerState.NormalizedTime.

It also has a non-serialized OnEnd callback which is assigned to AnimancerState.OnEnd. This can be used to easily cache a delegate so that the same instance is used every time the animation is played instead of allocating garbage every time or needing to add your own field to cache it.
AnimancerTransition

A ScriptableObject which has a single ClipState.Serializable field, allowing you to define those details as an asset that can be shared in multiple places throughout your project rather than as part of a specific object in a scene. You can create a transition asset using the Assets menu: Assets/Create/Animancer/Transition.

You can add more parameters to a transition by inheriting from AnimancerTransition or create transitions for other state types by inheriting from AnimancerTransition<TState>.
ClipState
.SerializableWithEndEvent

Inherits from ClipState.Serializable to add a UnityEvent that it uses as the OnEnd callback so it can be configured in the Inspector.

If you have UltEvents (also made by Kybernetik like Animancer), you could make a serializable which instead has an UltEvent to make use of their advantages over standard UnityEvents.

As mentioned in the Basics/Playing and Fading example, one of the advantages is the ability to call the AnimancerComponent.Play and CrossFade methods which UnityEvents can't call (since they have non-void return types and multiple parameters).

using Animancer;
using System;
using UltEvents;
using UnityEngine;

[Serializable]
public class UltClipStateSerializable : ClipState.Serializable, ISerializationCallbackReceiver
{
    [SerializeField]
    private UltEvent _OnEnd;

    public new UltEvent OnEnd
    {
        get { return _OnEnd; }
        set
        {
            _OnEnd = value;
            base.OnEnd = value.Invoke;
        }
    }

    public virtual void OnBeforeSerialize() { }

    public virtual void OnAfterDeserialize()
    {
        base.OnEnd = _OnEnd.Invoke;
    }
}

Controller Serializables

Type Details
ControllerState
.Serializable<TState>		 abstract base class for serializables that create states derived from ControllerState which play Animator Controllers. Only has a RuntimeAnimatorController field.
ControllerState
.Serializable

Inherits from ControllerState.Serializable<ControllerState> to create a base ControllerState.
FloatControllerState
.Serializable

Inherits from ControllerState.Serializable<FloatControllerState> and adds a ParameterName field to create a FloatControllerState which wraps that particular parameter. The field uses a dropdown menu in the Inspector to allow you to select the name of any Float parameter.

Similar serializables exist for Vector2ControllerState and Vector3ControllerState.

Type Details
ManualMixerState
.Serializable<TMixer>

abstract base class for serializables that create states derived from ManualMixerState. Has an AnimationClip array and Speeds array.

These serializables all use a reorderable list to configure their states. You can Right Click on the list header to open a context menu with several useful functions.

Inherits from ManualMixerState.Serializable<ManualMixerState> to create a base ManualMixerState.

Inherits from MixerState.Serializable<LinearMixerState,float> to create a LinearMixerState.

Inherits from MixerState.Serializable<MixerState<Vector2>,Vector2> to create a MixerState<Vector2>.

Custom Serializables

You can create your own serializables by implementing IAnimancerTransition from scratch or inheriting from AnimancerState.Serializable or any of its derived types. When doing so, you may also want to alter the way it is drawn in the Inspector by creating another class that inherits from the Drawer class inside the Serializable you are inheriting from or implementing your own PropertyDrawer.

  • The Basics/Sequence Coroutine example implements a class that inherits from ClipState.Serializable to add a PlayDuration field to determine how long a looping animation should be played during the sequence.
  • The Locomotion/Root Motion example implements a class that inherits from ClipState.Serializable to add an ApplyRootMotion field to determine whether the Animator.applyRootMotion should be enabled for each animation.
