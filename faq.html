Frequently Asked Questions

Question Answer
Why replace Mecanim? The Mecanim vs. Animancer section explains the problems with Mecanim and the ways Animancer solves them.
What's the difference between Pro and Lite? Animancer Pro includes more features which the Lite version only includes in the Unity Editor for trial purposes. See the Feature Comparison for the actual list of differences.
Does it come with the source code? Animancer Pro includes the full source code. The DLLs in the package are empty dummies to overwrite the ones from the Lite version to allow seamless upgrades.
Which version of Unity is recommended? Minimum 2017.1. Recommended 2018.3. Tested up to 2019.2 (beta). See Unity Versions for more details.
What is the performance like? Animancer is notably faster at Instantiation and very slightly faster in Average Frame Rate, though the differences are small enough not to matter. See the Performance Benchmarks for more details.
How can I control the speed or time of an animation?
[Pro-Only] Use the AnimancerState returned by the Play or CrossFade method: (click to expand)
void PlayAnimation(AnimancerComponent animancer, AnimationClip clip)
{
    var state = animancer.Play(clip);
    state.Time = ...
    state.NormalizedTime = ...
    state.Speed = ...
    state.OnEnd = ...
}
Can I still use Animator Controllers? Yes, you can use Controller States to play them inside Animancer so that you can mix them with separate animations as necessary. You can even blend between multiple Animator Controllers if you want. The Locomotion/Linear Blending example demonstrates how to play an Animator Controller containing a Blend Tree and control its parameter.
