Frequently Asked Questions

Question Answer
Help? Feel free to post in the Unity forum thread or email AnimancerUnityPlugin@gmail.com if you have any questions.
Why replace Mecanim? The Mecanim vs. Animancer section explains the problems with Mecanim and the ways Animancer solves them.
What about Playables? Unity's Playables API is the low level system which both Mecanim and Animancer are built upon. As mentioned above the Feature Comparison table, it allows you to build animation systems but is too low level to be easily useful on its own.
What's the difference between Pro and Lite? Animancer Pro includes more features which the Lite version only includes in the Unity Editor for trial purposes. See the Feature Comparison for the actual list of differences.
Does it come with the source code? Animancer Pro includes the full source code as regular C# scripts. The DLLs in the package are empty dummies to overwrite the ones from the Lite version to allow seamless upgrades. Animancer Lite only includes the source code of some parts such as the Finite State Machine system, but not the core of Animancer.
Which version of Unity is recommended? Minimum 2017.1. Recommended 2018.3. Tested up to 2019.2 (beta). See Unity Versions for more details.
Is it compatible with X? Probably. Animancer is based on the Playables API, which means most systems developed for regular Mecanim Animator Controllers should work normally alongside Animancer, with a few exceptions. See the Compatability With Other Systems section for more details.
Is the performance good? Yes. Animancer is notably faster than Mecanim at Instantiation and very slightly faster in Average Frame Rate, though the differences are small enough not to matter. See the Performance Benchmarks for more details.
How can I control the speed or time of an animation?
[Pro-Only] Use the AnimancerState returned by the Play or CrossFade method: (click to expand)
void PlayAnimation(AnimancerComponent animancer, AnimationClip clip)
{
    var state = animancer.Play(clip);
    state.Time = ...
    state.NormalizedTime = ...
    state.Speed = ...
    state.OnEnd = ...
}

Speed and time control are demonstrated in the Fine Control examples.

Those properties are not included as parameters in the Play and CrossFade methods because they already have multiple overloads with several default values and a call like CrossFade(clip, 0.25f, 1, 0.5f, 0.5f) would not be particularly clear about what each value does without close examination. But if you really wanted to, you could implement them using extension methods:

public static class AnimancerExtensions
{
    public static AnimancerState Play(this AnimancerComponent animancer, AnimationClip clip, int layer, float time, float speed, System.Action onEnd)
    {
        var state = animancer.Play(clip, layer);
        state.Time = time;
        state.Speed = speed;
        state.OnEnd = onEnd;
        return state;
    }
}
Can I still use Animator Controllers? [Pro-Only] Yes, you can use HybridAnimancerComponents and Controller States to play them inside Animancer so that you can mix them with separate animations as necessary. You can even blend between multiple Animator Controllers if you want. The Animator Controller examples demonstrate how to do so.
Can I mirror animations? As explained in the Feature Comparison table, you can mirror Humanoid animations in their import settings but not per state. If you need both mirrored and un-mirrored versions of a particular animation, you will need to duplicate it and mirror one of them.
