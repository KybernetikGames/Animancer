Fading

This page needs to be updated for Animancer v4.0.

Calling AnimancerComponent.Play will immediately stop all other animations and snap the character's pose to the start the new one. However, it is often desirable to transition smoothly between animations by fading the new one in while fading the old one(s) out over a short period of time, giving the impression of smooth motion without needing to create additional transition animations. This is known as "Cross Fading" due to the way the animation weights cross over each other.

By default, a fade will take 0.25 seconds to complete. Animancer Pro allows you to specify a custom duration (Animancer Lite allows you to try custom durations in the Unity Editor, but it will always use the default in runtime builds). See the Feature Comparison for more information.

Animancer has several different ways of fading between animations:

Method Effect
AnimancerComponent
.CrossFade

Fades out all of the currently active animations (towards Weight = 0 so they don't control the model at all) over a specified period of time while fading in the specified animation (towards Weight = 1 so it fully controls the model). If the animation is already playing, it will continue to do so from the current time.

void CrossFadeExample(AnimancerComponent animancer, AnimationClip clip)
{
    // Default 0.25 seconds fade duration.
    animancer.CrossFade(clip);

    // Fade over 0.8 seconds.
    animancer.CrossFade(clip, 0.8f);

    // Fade for 20% of the new clip's duration.
    animancer.CrossFade(clip, clip.length * 0.2f);

    // Fade for 10% of the old clip's duration.
    animancer.CrossFade(clip, animancer.CurrentState.Length * 0.1f);
}

The Basics/Playing and Fading example demonstrates the use of this feature in more detail.
AnimancerComponent
.CrossFadeFromStart

Like CrossFade, except that if the animation is already playing, it will be faded out while a new instance of the same animation fades in from the start of the clip. This ensures that it will always smoothly transition from the current pose into the start of the animation.

The Basics/Playing and Fading example demonstrates the use of this feature in more detail.
AnimancerComponent
.Transition

This method takes an IAnimancerTransition which contains the details of a state and the way it should be played. It uses these details to automatically choose between Play, CrossFade, and CrossFadeFromStart. This method is most commonly used with Transitions to allow you to configure the details in the Inspector instead of hard coding them in a script.

AnimancerNode
.StartFade

Starts fading a specific node without affecting any others (AnimancerLayer and AnimancerState both inherit from AnimancerNode). It is commonly used to fade Layers out when they aren't in use. There is no CrossFade for layers because they are not commonly used in that manner, but you can achieve the same thing by calling StartFade(0) to fade some layers out and StartFade(1) to fade others in.

void StartFadeExample(AnimancerComponent animancer, AnimationClip clip)
{
    const float FadeDuration = AnimancerPlayable.DefaultFadeDuration;

    // Fade an animation in on Layer 1.
    state = animancer.CrossFade(clip, FadeDuration, 1);

    // Then when it finishes, fade that layer out.
    state.OnEnd = () => animancer.GetLayer(1).StartFade(0);
}
