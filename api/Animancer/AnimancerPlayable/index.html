AnimancerPlayable Class

Summary

A UnityEngine.Playables.PlayableBehaviour which can be used as a substitute for the UnityEngine.RuntimeAnimatorController normally used to control an UnityEngine.Animator.

This class can be used as a custom yield instruction to wait until all animations finish playing.

CrossFade(AnimancerState, float) AnimancerState
Starts fading in the 'state' over the course of the 'fadeDuration' while fading out all others in the same layer. Returns the 'state'.

If the 'state' was already playing, it will continue doing so from the current time, unlike Animancer.AnimancerPlayable.CrossFadeFromStart(Animancer.AnimancerState,System.Single).

If the 'state' was already playing and fading in with less time remaining than the 'fadeDuration', this method will allow it to complete the existing fade rather than starting a slower one.

If the layer currently has 0 Animancer.AnimancerNode.Weight, this method will instead start fading in the layer itself and simply Animancer.AnimancerState.Play the 'state'.

Animancer Lite only allows the default 'fadeDuration' (0.3 seconds) in a runtime build.

CrossFadeFromStart(AnimancerState, float) AnimancerState
Starts fading in the 'state' from the start over the course of the 'fadeDuration' while fading out all others in the same layer. Returns the 'state'.

If the 'state' isn't currently at 0 Animancer.AnimancerNode.Weight, this method will actually fade it to 0 along with the others and create and return a new state with the same clip to fade to 1. This ensures that calling this method will always fade out from all current states and fade in from the start of the desired animation. States created for this purpose are cached so they can be reused in the future.

Calling this method repeatedly on subsequent frames will probably have undesirable effects; you most likely want to use Animancer.AnimancerPlayable.CrossFade(Animancer.AnimancerState,System.Single) instead.

If the layer currently has 0 Animancer.AnimancerNode.Weight, this method will instead start fading in the layer itself and simply Animancer.AnimancerState.Play the 'state'.
