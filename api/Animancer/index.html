Animancer Namespace

Summary

Everything in the Animancer plugin is inside this namespace. Examples of how to use the Animancer plugin.

Class Types

Class Summary
AnimancerController A substitute for a UnityEngine.RuntimeAnimatorController which allows you to freely play animations on an UnityEngine.Animator without using the standard Mecanim state machine system.

This class can be used as a custom yield instruction to wait until all animations finish playing.
AnimancerExtensions Various extension methods.
AnimancerLayer A layer on which animations can play with their states managed independantly of other layers while blending the output with those layers.

This class can be used as a custom yield instruction to wait until all animations finish playing.
AnimancerPlayable A UnityEngine.Playables.PlayableBehaviour which can be used as a substitute for the UnityEngine.RuntimeAnimatorController normally used to control an UnityEngine.Animator.

This class can be used as a custom yield instruction to wait until all animations finish playing.
AnimancerState Base class for all states in an Animancer.AnimancerPlayable graph. Each state is a wrapper for a UnityEngine.Playables.Playable in the UnityEngine.Playables.PlayableGraph.

This class can be used as a custom yield instruction to wait until the animation either stops playing or reaches its end.
AnimancerTransition A UnityEngine.ScriptableObject which defines various details about how to play an animation so they can be configured in the Unity Editor instead of hard coded.

To trigger a transition, simply pass it as a parameter into the Animancer.AnimancerController Play or CrossFade methods.
AnimationMixer [Pro-Only] Base class for Animancer.AnimancerStates which blend other states together.
AnimationMixer<TParameter> [Pro-Only] Base class for Animancer.AnimationMixers which blend an array of Animancer.AnimationMixer`1.States together based on a Animancer.AnimationMixer`1.Parameter.
AnimatorControllerState [Pro-Only] An Animancer.AnimancerState which plays a UnityEngine.RuntimeAnimatorController. You can control this state very similarly to an UnityEngine.Animator via its Animancer.AnimatorControllerState.Playable field.
CartesianAnimationMixer [Pro-Only] An Animancer.AnimancerState which blends an array of other states together based on a two dimensional parameter and thresholds using Gradient Band Interpolation.

This mixer type is similar to the 2D Freeform Cartesian Blend Type in Mecanim Blend Trees.
ClipState An Animancer.AnimancerState which plays an UnityEngine.AnimationClip.
DirectionalAnimationMixer [Pro-Only] An Animancer.AnimancerState which blends an array of other states together based on a two dimensional parameter and thresholds using Polar Gradient Band Interpolation.

This mixer type is similar to the 2D Freeform Directional Blend Type in Mecanim Blend Trees.
EventfulAnimancerController Base class for Animancer.AnimancerControllers which need to do something whenever an animation is played. It adds an Animancer.EventfulAnimancerController.OnBeforePlay method and overrides all the usual Play and CrossFade methods to call it before performing their regular function.
FadeExtensions Extension methods for Animancer.ICanFade.
FloatParameterControllerState [Pro-Only] An Animancer.AnimatorControllerState which manages a float parameter.
LegacyAnimancerController An Animancer.AnimancerController which uses the UnityEngine.Object.names of UnityEngine.AnimationClips to register them in its dictionary so they can be referenced using strings as well as the clips themselves.

It also has fields to automatically register animations on startup and play the first one automatically without needing another script to control it, much like Unity's Legacy UnityEngine.Animation component.
LinearAnimationMixer [Pro-Only] An Animancer.AnimancerState which linearly blends an array of other states together.

This mixer type is similar to the 1D Blend Type in Mecanim Blend Trees.
SimpleEventAnimancerController An Animancer.AnimancerController which uses a C# delegate to react to animation events called "Event".
Vector2ParameterControllerState [Pro-Only] An Animancer.AnimatorControllerState which manages two float parameters.
Vector3ParameterControllerState [Pro-Only] An Animancer.AnimatorControllerState which manages three float parameters.
