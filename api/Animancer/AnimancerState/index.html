AnimancerState Class

Summary

Base class for all states in an Animancer.AnimancerPlayable graph. Each state is a wrapper for a UnityEngine.Playables.Playable in the UnityEngine.Playables.PlayableGraph.

This class can be used as a custom yield instruction to wait until the animation either stops playing or reaches its end.

    IEnumerator, IDisposable

Remarks

There are various different ways of getting a state:
  • Use one of the state's constructors. Generally the first parameter is a layer or mixer which will be used as the state's parent. If not specified, you will need to call SetParent manually. Also note than an AnimancerController can be implicitly cast to its first layer.
  • CreateState creates a new ClipState. You can optionally specify a custom 'key' to register it in the dictionary instead of the default (the 'clip' itself).
  • GetOrCreateState looks for an existing state registered with the specified 'key' and only create a new one if it doesn’t already exist.
  • GetState returns an existing state registered with the specified 'key' if there is one.
  • TryGetState is similar but returns a bool to indicate success and returns the 'state' as an out parameter.
  • Play and CrossFade also return the state they play.

Note that when inheriting from this class, the Animancer.AnimancerState._BasePlayable field must be assigned in the constructor to avoid throwing System.ArgumentExceptions throughout the system.

Events

Name Type Summary
OnEnd Action
A callback which is triggered when the UnityEngine.AnimationClip reaches its end (not when the state is exited). If the clip loops, this event will be triggered every time it passes the end of the clip.

This event is automatically cleared by Animancer.AnimancerState.Play and Animancer.AnimancerState.StartFade(System.Single,System.Single).
