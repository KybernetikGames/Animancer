LegacyAnimancerController Class

Summary

An Animancer.AnimancerController which uses the UnityEngine.Object.names of UnityEngine.AnimationClips to register them in its dictionary so they can be referenced using strings as well as the clips themselves.

It also has fields to automatically register animations on startup and play the first one automatically without needing another script to control it, much like Unity's Legacy UnityEngine.Animation component.

[AddComponentMenu("Animancer/Legacy Animancer Controller")]

public class LegacyAnimancerController : AnimancerController, IAnimancer, 
    IEnumerable<AnimancerState>, IEnumerable, IEnumerator

Remarks

Despite the word 'Legacy' in the name, there are no plans to deprecate this component in the forseeable future. The name was simply chosen for its similarities with Unity's Legacy UnityEngine.Animation component.

ForceSetClip(object, AnimationClip, int) AnimancerState
If a state is registered with the 'key', this method verifies that it has the correct 'clip' and returns it. If the Animancer.AnimancerState.Clip doesn't match the specified 'clip', the state is destroyed and a new one is created in its place with the correct 'clip'. A new one is also created if no existing state was registered with the 'key'.

Note that destroying and creating states is more computationally expensive than most other operations and should be avoided at performance intensive times. Consider using Animancer.AnimancerController.GetOrCreateState(UnityEngine.AnimationClip,System.Int32) instead, which will throw an exception instead of replacing the state.

Determines a key for the clip using the Animancer.AnimancerController.GetKey(UnityEngine.AnimationClip) method (which uses the 'clip' itself unless that method is overridden).
